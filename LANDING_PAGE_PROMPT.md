**FASE 02: PERBANDINGAN BERPASANGAN AHP & EKSTRAKSI VEKTOR EIGEN**

**Penjelasan untuk Landing Page**:
"Manusia kesulitan mendistribusikan 100 poin bobot secara akurat. AHP memecah kompleksitas menjadi keputusan biner sederhana."

**Skala Saaty 1-9: Translasi Preferensi ke Angka**
```
1 = Sama pentingnya (Equally preferred)
3 = Sedikit lebih penting (Moderately preferred)
5 = Lebih penting secara kuat (Strongly preferred)
7 = Sangat kuat (Very strongly preferred)
9 = Kepentingan ekstrem (Extremely preferred)
```

**Keunggulan Skala Saaty vs Likert**:
- Likert: Pengukuran absolut, rentan central tendency bias
- Saaty: Pengukuran relatif, memaksa komparasi objektif
- Likert: Subjektif per item
- Saaty: Objektif melalui perbandingan berpasangan

**Proses Matematis**:
1. User membandingkan kriteria secara berpasangan (n kriteria = n(n-1)/2 perbandingan)
2. Sistem membentuk matriks resiprokal positif (n×n)
3. Aturan resiprokal: Jika A/B = 3, maka B/A = 1/3
4. Diagonal matriks = 1 (kriteria vs dirinya sendiri)
5. Ekstraksi Principal Eigenvector melalui normalisasi matriks
6. Output: Vektor bobot prioritas yang matematis valid

**Formula Kunci**:
```
A × w = λmax × w
```
Dimana:
- A = Matriks perbandingan
- w = Vektor bobot (yang dicari)
- λmax = Nilai eigen utama

**Visual**: 
- Interactive pairwise comparison simulator
- Matrix visualization dengan color coding
- Eigenvector extraction animation


**FASE 03: VALIDASI RASIO KONSISTENSI (CR) - AUDITOR MATEMATIS**

**Penjelasan untuk Landing Page**:
"Sistem tidak menerima input mentah. Setiap keputusan Anda diaudit untuk memastikan koherensi logis."

**Problem: Inkonsistensi Manusia**
Contoh paradoks:
- User: "Peluang Karir > Biaya Kuliah" ✓
- User: "Biaya Kuliah > Jarak Kampus" ✓
- User: "Jarak Kampus > Peluang Karir" ✗ (KONTRADIKSI!)

Logika transitif: Jika A>B dan B>C, maka A harus >C

**Solusi: Consistency Ratio (CR)**

**Formula Perhitungan**:
```
CI = (λmax - n) / (n - 1)
CR = CI / RI
```

Dimana:
- CI = Consistency Index
- λmax = Nilai eigen utama
- n = Dimensi matriks
- RI = Random Index (dari simulasi ribuan matriks acak)

**Ambang Batas Validasi**:
- CR ≤ 0.1 (10%) → VALID ✓ Lanjut ke fase berikutnya
- CR > 0.1 (10%) → INVALID ✗ Sistem menolak, minta kalibrasi ulang

**Mengapa Ini Penting?**:
- Mencegah "garbage in, garbage out"
- Memaksa user merenungkan sistem nilai internal
- Menjamin output rekomendasi berakar pada logika rasional
- Fungsi pedagogis: Edukasi tentang konsistensi berpikir

**Adaptive Recalibration**:
Jika CR > 0.1, sistem memberikan:
1. Feedback spesifik tentang inkonsistensi
2. Saran perbaikan perbandingan
3. Visualisasi kontradiksi
4. Kesempatan revisi tanpa restart

**Visual**:
- CR meter dengan zona hijau/merah
- Inconsistency heatmap pada matriks
- Before/after comparison setelah kalibrasi


**FASE 04: NORMALISASI MATRIKS TOPSIS & EVALUASI KINERJA EMPIRIS**

**Penjelasan untuk Landing Page**:
"Dari psikologi subjektif menuju realitas objektif. TOPSIS mengevaluasi performa aktual setiap jurusan."

**Konstruksi Matriks Keputusan (m×n)**
- m = Jumlah alternatif jurusan (Teknik Informatika, Ekonomi, Sosiologi, dll)
- n = Jumlah kriteria evaluasi

**Tipologi Kriteria**:

1. **Benefit Criteria** (Semakin tinggi = semakin baik)
   - Nilai Akademik
   - Akreditasi Institusi (BAN-PT)
   - Kemudahan Kurikulum
   - Prestasi Akademik
   - Reputasi Alumni
   - Peluang Karir
   - Kerjasama Industri

2. **Cost Criteria** (Semakin rendah = semakin baik)
   - Biaya Kuliah (Tuition Fees)
   - Jarak Geografis
   - Tingkat Persaingan Jurusan
   - Historical Drop-out Rate
   - Masa Tunggu Kerja (dari Tracer Study)

**Proses Normalisasi Vektor**:

**Problem**: Data beroperasi pada skala berbeda
- Biaya kuliah: Jutaan Rupiah
- Tingkat persaingan: Rasio desimal 0-1
- Skor tes: Ratusan poin
→ Tidak bisa dibandingkan langsung!

**Solusi**: Vector Normalization
```
rij = xij / √(Σ xij²)
```

Transformasi setiap nilai menjadi format standar tak berdimensi.

**Weighted Normalized Matrix**:
Menggabungkan output AHP (Fase 02) dengan matriks ternormalisasi:
```
vij = wi × rij
```

**Output Fase 04**:
Matriks hibrida yang merepresentasikan:
- Kinerja objektif setiap program studi
- Diskalakan berdasarkan prioritas psikologis individual
- Siap untuk perhitungan jarak geometris

**Visual**:

- Normalization transformation animation
- Weighted matrix heatmap


**FASE 05: PENCARIAN JARAK SOLUSI IDEAL - GEOMETRI KEPUTUSAN**

**Penjelasan untuk Landing Page**:
"Alternatif terbaik = Terdekat dari kesempurnaan, terjauh dari kegagalan."

**Premis Geometris TOPSIS**:
Dalam ruang multidimensi, setiap jurusan memiliki posisi relatif terhadap dua kutub ekstrem.

**Definisi Kutub Matematika**:

1. **Positive Ideal Solution (A+)**
   - Jurusan hipotetis "sempurna"
   - Nilai tertinggi semua Benefit criteria
   - Nilai terendah semua Cost criteria
   - Contoh: Akreditasi A++, Biaya Rp 0, Peluang Karir 100%, Persaingan 0%
   - Tidak perlu exist di realitas (jangkar matematis)

2. **Negative Ideal Solution (A-)**
   - Jurusan hipotetis "terburuk"
   - Nilai terendah semua Benefit criteria
   - Nilai tertinggi semua Cost criteria
   - Contoh: Tidak terakreditasi, Biaya maksimal, Peluang Karir 0%, Persaingan ekstrem

**Kalkulasi Jarak Euclidean**:

**Jarak ke Ideal Positif (Si+)**:
```
Si+ = √[Σ(vij - vj+)²]
```

**Jarak ke Ideal Negatif (Si-)**:
```
Si- = √[Σ(vij - vj-)²]
```

**Mengapa Euclidean vs Hamming?**
- Euclidean: Mengukur besaran spasial sejati dalam ruang n-dimensi
- Hamming: Hanya divergensi linear (terbukti 91-93% inkompatibilitas)
- Euclidean: Lebih akurat untuk data kontinu multidimensi

**Interpretasi Geometris**:
- Si+ kecil → Jurusan mendekati "kesempurnaan"
- Si- besar → Jurusan menjauh dari "kegagalan"
- Kombinasi keduanya → Pemeringkatan final

**Visual**:
- 3D spatial visualization (simplified dari n-dimensi)
- Distance vector animation
- Ideal solution positioning diagram


**FASE 06: KEPUTUSAN FINAL & PEMERINGKATAN ADAPTIF**

**Penjelasan untuk Landing Page**:
"Satu angka yang merangkum ribuan perhitungan. Rekomendasi yang 100% dijamin rasionalitasnya."

**Closeness Coefficient (Ci)**:
```
Ci = Si- / (Si+ + Si-)
```

**Rentang Nilai**:
- Ci = 1.0 → Identik dengan Solusi Ideal Positif (sempurna)
- Ci = 0.0 → Identik dengan Solusi Ideal Negatif (terburuk)
- 0 < Ci < 1 → Semua alternatif real berada di sini

**Pemeringkatan Final**:
Semua jurusan diurutkan berdasarkan Ci (descending):
1. Rank 1: Ci tertinggi → Rekomendasi utama
2. Rank 2: Ci kedua → Alternatif backup
3. Rank 3-5: Opsi cadangan
4. Rank rendah: Tidak direkomendasikan

**Jaminan Kualitas Rekomendasi**:
✓ Diaudit psikologis (CR < 0.1)
✓ Dinormalisasi ganda (eliminasi bias skala)
✓ Divalidasi dengan data BAN-PT
✓ Diselaraskan dengan Tracer Study
✓ Bebas dari bias kognitif
✓ Bebas dari groupthink
✓ Bebas dari manipulasi emosional

**Output untuk User**:
- Top 5 rekomendasi dengan skor Ci
- Breakdown kontribusi setiap kriteria
- Comparison matrix antar top recommendations
- Reasoning explanation (mengapa jurusan X direkomendasikan)
- Risk assessment untuk setiap pilihan

**Visual**:
- Ranking leaderboard dengan Ci scores
- Contribution breakdown pie chart
- Side-by-side comparison table
- Decision confidence meter


### SECTION 5: ANALISIS KOMPARATIF - "MENGAPA AHP-TOPSIS SUPERIOR"
**Objective**: Establish competitive advantage through data

**Comparison Matrix: AHP-TOPSIS vs Algoritma Lain**

| Parameter                | SAW/SMART          | Profile Matching   | TOPSIS Murni    | AHP-TOPSIS               |
| --------------------------| --------------------| --------------------| -----------------| --------------------------|
| **Metode Bobot**         | Manual/Statis      | Gap Core/Secondary | Pre-defined     | Eigenvector              |
| **Mitigasi Bias**        | Tidak ada          | Rendah             | Tidak ada       | Sangat Tinggi (CR<0.1)   |

| **Kompleksitas Data**    | Rendah             | Sedang             | Tinggi          | Sangat Tinggi            |

**Kelemahan Fatal Metode Konvensional**:

1. **SAW (Simple Additive Weighting)**
   - Bobot ditentukan arbitrary oleh admin
   - Tidak ada mekanisme validasi preferensi user
   - Rentan distorsi oleh data outlier
   - Akurasi hanya 68.64% untuk kasus kompleks

2. **SMART (Simple Multi-Attribute Rating)**
   - Bobot universal statis (tidak personalized)
   - Tidak menangkap variasi profil individual
   - Tidak ada error handling untuk kontradiksi
   - Gagal pada kasus edge case

3. **Profile Matching**
   - Hanya cocok untuk single-target qualification
   - Kesulitan dengan matriks multidimensi dinamis
   - Akurasi standalone hanya 45%
   - Tidak scalable untuk ratusan alternatif

4. **TOPSIS Standalone**
   - Tidak memiliki mekanisme pembobotan psikometrik
   - Bergantung pada bobot pre-defined (subjektif)
   - Tidak ada audit konsistensi preferensi
   - Akurasi 73% (baik, tapi tidak optimal)

**Keunggulan Hibrida AHP-TOPSIS**:
✓ AHP menambal kelemahan TOPSIS dalam pembobotan
✓ TOPSIS menambal kelemahan AHP dalam skalabilitas
✓ Sinergi menghasilkan akurasi 95.71%
✓ Validasi ganda: CR audit + normalisasi vektor
✓ Terbukti dalam blackbox testing operasional

### SECTION 7: TRANSPARANSI KETERBATASAN - "INTELLECTUAL HONESTY"
**Objective**: Build trust through transparency

**Batasan Metodologis yang Harus Dikomunikasikan**:

**1. Asumsi Determinisme Statis**
- **Limitasi**: Algoritma mengasumsikan data lingkungan bersifat statis
- **Realitas**: Preferensi manusia bersifat fluid, metrik institusional berfluktuasi
- **Mitigasi**: Sistem menyarankan re-assessment berkala (setiap 6-12 bulan)
- **Impact**: Jika user salah input pada Fase 01, seluruh perhitungan tetap matematis valid tapi esensial cacat

**2. Rank Reversal (Pembalikan Peringkat)**
- **Limitasi**: Penambahan/penghapusan alternatif dapat mengubah peringkat relatif
- **Penyebab**: Normalisasi vektor bergantung pada keseluruhan dataset
- **Mitigasi**: TOPSIS termodifikasi + AHP eigen menurunkan probabilitas signifikan
- **Transparansi**: Sistem menampilkan "confidence interval" untuk top 3 rekomendasi

**3. Beban Kognitif Perbandingan Berpasangan**
- **Limitasi**: n kriteria = n(n-1)/2 perbandingan
- **Contoh**: 10 kriteria = 45 perbandingan → Decision Fatigue
- **Mitigasi**: 
  - Sistem membatasi maksimal 7 kriteria utama (sweet spot)
  - Progress indicator untuk motivasi
  - Save & resume functionality
  - Adaptive questioning (skip jika CR sudah optimal)

**4. Kualitas Data Input**
- **Limitasi**: "Garbage in, garbage out" tetap berlaku
- **Dependency**: Akurasi BAN-PT dan Tracer Study data
- **Mitigasi**:
  - Data validation layer
  - Multiple source verification
  - Timestamp freshness indicator
  - User feedback loop untuk koreksi

**Komitmen Transparansi**:
✓ Sistem menampilkan "confidence score" untuk setiap rekomendasi
✓ Breakdown reasoning: "Mengapa jurusan X direkomendasikan"
✓ Sensitivity analysis: "Bagaimana jika kriteria Y berubah?"
✓ Disclaimer: "Rekomendasi adalah guidance, bukan keputusan final"
✓ Encourage: Konsultasi dengan konselor profesional tetap penting

### SECTION 10: HOW IT WORKS - "USER JOURNEY"
**Objective**: Simplify complexity for user understanding

**Step-by-Step Process**:

**STEP 1: Registrasi & Profiling (15 menit)**
- Buat akun dengan email/Google
- Isi data dasar: Nama, asal sekolah, jurusan SMK/SMA
- Lengkapi profil akademik: Nilai rata-rata, prestasi
- Upload dokumen pendukung (opsional)

**STEP 2: Asesmen Psikologis (20 menit)**
- Tes Minat Intrinsik: 30 pertanyaan tentang preferensi subjek
- Tes Kemampuan Logika: 20 soal reasoning & spatial
- Tes Preferensi Lingkungan: 15 skenario pembelajaran
- Sistem menganalisis gap antara profil Anda vs tuntutan jurusan

**STEP 3: Perbandingan Berpasangan AHP (10 menit)**
- Sistem menyajikan kriteria yang relevan dengan profil Anda
- Anda membandingkan kriteria secara berpasangan (contoh: "Peluang Karir vs Biaya Kuliah")
- Gunakan slider 1-9 untuk menunjukkan tingkat kepentingan
- Sistem real-time menghitung Consistency Ratio
- Jika CR > 0.1, sistem meminta kalibrasi ulang dengan guidance

**STEP 4: Preferensi Tambahan (5 menit)**
- Lokasi geografis: Dalam kota, luar kota, luar pulau
- Budget range: <5jt, 5-10jt, 10-20jt, >20jt per semester
- Tipe institusi: PTN, PTS, Politeknik, Institut
- Preferensi khusus: Beasiswa, asrama, program internasional

**STEP 5: Komputasi Algoritma (30 detik)**
- Sistem memproses 6 fase AHP-TOPSIS
- Mengintegrasikan data BAN-PT & Tracer Study
- Menghitung jarak Euclidean ke Ideal Solution
- Menghasilkan Closeness Coefficient untuk setiap alternatif
- Progress bar menampilkan tahapan komputasi

**STEP 6: Hasil & Rekomendasi (Unlimited)**
- Top 5 rekomendasi jurusan dengan skor Ci
- Breakdown detail setiap rekomendasi:
  - Mengapa jurusan ini cocok untuk Anda
  - Kekuatan & kelemahan relatif
  - Prospek karir & gaji rata-rata
  - Universitas yang menawarkan (dengan akreditasi)
- Comparison matrix: Bandingkan top 3 side-by-side
- Download PDF report lengkap
- Share dengan orang tua/konselor

**STEP 7: Konsultasi Lanjutan (Opsional)**
- Book session dengan konselor BK bersertifikat
- Diskusi hasil dengan konteks personal

### SECTION 12: FAQ - "JAWABAN UNTUK KERAGUAN"
**Objective**: Address objections preemptively

**Q: Apakah sistem ini bisa 100% menjamin saya tidak akan menyesal?**
A: Tidak ada sistem yang bisa menjamin 100%, termasuk intuisi Anda sendiri. Yang kami jamin adalah: rekomendasi kami didasarkan pada 95.71% akurasi historis, validasi matematis ketat (CR<0.1), dan data objektif dari BAN-PT & Tracer Study. Ini jauh lebih reliable daripada "ikut teman" atau "feeling".

**Q: Bagaimana jika saya tidak setuju dengan rekomendasi sistem?**
A: Sistem adalah decision support, bukan decision maker. Anda tetap pemegang keputusan final. Namun, kami sarankan untuk memahami reasoning di balik rekomendasi sebelum mengabaikannya. Sering kali, sistem menemukan insight yang tidak terlihat secara intuitif.

**Q: Apakah data saya aman?**
A: Sangat aman. Kami menggunakan enkripsi end-to-end, compliance dengan UU Perlindungan Data Pribadi, dan tidak pernah menjual data ke pihak ketiga. Data Anda hanya digunakan untuk menghasilkan rekomendasi personal.

**Q: Berapa lama proses asesmen?**
A: Total sekitar 50 menit untuk asesmen lengkap. Anda bisa pause dan resume kapan saja. Komputasi algoritma hanya butuh 30 detik.

**Q: Apakah sistem ini cocok untuk siswa SMA IPA/IPS juga?**
A: Absolut! Sistem dirancang untuk semua latar belakang: SMK, SMA IPA, SMA IPS, MA. Algoritma akan menyesuaikan kriteria evaluasi berdasarkan profil Anda.

**Q: Bagaimana jika saya ingin ganti jurusan setelah kuliah?**
A: Anda bisa re-assessment kapan saja (paket Student/Premium). Sistem akan memberikan rekomendasi pivot berdasarkan kondisi terbaru Anda.

**Q: Apakah rekomendasi hanya untuk PTN atau termasuk PTS?**
A: Database kami mencakup PTN, PTS, Politeknik, dan Institut. Anda bisa filter berdasarkan preferensi.

**Q: Bagaimana sistem menangani jurusan baru yang belum ada data Tracer Study?**
A: Untuk jurusan emerging (AI, Blockchain, dll), sistem menggunakan proxy data dari jurusan terdekat dan industry trend analysis. Kami transparant menampilkan "confidence level" untuk setiap rekomendasi.

**Q: Apakah ada batasan usia atau tahun kelulusan?**
A: Tidak ada batasan. Sistem cocok untuk siswa kelas 12, gap year, atau bahkan profesional yang ingin kuliah lagi.

## COPYWRITING GUIDELINES

### TONE OF VOICE
- **Authoritative yet Accessible**: Gunakan terminologi ilmiah, tapi jelaskan dengan analogi sederhana
- **Data-Driven**: Setiap klaim harus backed by statistik atau penelitian
- **Empathetic**: Acknowledge anxiety siswa tentang pemilihan jurusan
- **Action-Oriented**: Setiap section berakhir dengan clear next step
- **Transparent**: Jujur tentang limitasi sistem

### FORMULA COPYWRITING

**AIDA Framework**:
1. **Attention**: Statistik mengejutkan tentang salah pilih jurusan
2. **Interest**: Penjelasan metodologi AHP-TOPSIS yang superior
3. **Desire**: Social proof, testimonial, impact stories
4. **Action**: Multiple CTA dengan urgency & risk reversal

**PAS Framework** (untuk problem sections):
1. **Problem**: Bias kognitif dalam pemilihan jurusan
2. **Agitate**: Konsekuensi jangka panjang dari keputusan salah
3. **Solve**: MajorMind sebagai solusi algoritmik

**FAB Framework** (untuk feature sections):
1. **Feature**: AHP Consistency Ratio validation
2. **Advantage**: Eliminasi kontradiksi logis
3. **Benefit**: Rekomendasi yang 100% rasional

### POWER WORDS TO USE
- Revolusioner, Algoritmik, Validasi, Presisi, Objektif
- Terbukti, Teruji, Akurat, Reliable, Kredibel
- Komprehensif, Mendalam, Holistik, Multidimensi
- Transformatif, Breakthrough, Cutting-edge, State-of-the-art
- Jaminan, Garansi, Terpercaya, Aman, Transparan

### WORDS TO AVOID
- "Mungkin", "Kira-kira", "Sepertinya" (terlalu uncertain)
- "Sempurna", "Terbaik di dunia" (overpromise)
- Jargon tanpa penjelasan
- Passive voice (gunakan active voice)
